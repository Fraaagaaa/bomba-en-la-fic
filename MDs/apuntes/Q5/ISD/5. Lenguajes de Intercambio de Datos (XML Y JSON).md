## Introducción
Queremos exponer la capa modelo creada en [[3. Diseño e implementación de la capa modelo]] como un servicio que pueda ser utilizado por aplicaciones remotas.
Las aplicaciones remotas:
- Pueden residir en otras máquinas o incluso acceder a la capa modelo a través de Internet.
- Pueden star escritas en cualquier lenguaje de programación.
- Proporcionarán su propia interfaz gráfica y pueden utilizar otras bases de datos o aplicaciones adicionales además de la nuestra.

Son necesarios leguajes o formatos de datos robustos, que permitan expresar y transmitir información compleja entre sistemas heretrogéneos.

## El lenguaje XML
### ¿Qué es XML?
XML (eXtensible Markup Language)
- Lenguaje de tags
- Estandarizado por el W3C
Es extensible
- XML no impone un conjunto de tags, sino sólo unas pocas normas sobre cómo usarlos.
	- Los tags se abren `(<tag>)` y se cierran `(</tag>)` y en medio pueden tener otros tags anidados.
	- Todos los elementos tienen un tag raíz
	- Los tags pueden tener atributos
Permite expresar información estructurada y fáciltmente parsable
Objetivo de este tema: Entender los fundamentos básicos de XML necesarios para esta asignatura
### Formato de un documento XML
**Documento XML**: Secuencia de caracteres que tiene texto en formato XML.
**Aplicación XML**: Conjunto particular de tags que permiten representar determinada información.
Se dice que un documento XML está **bien formado** si cumple con el conjunto de reglas que a continuación se expone.
Se distingue entre mayúsculas y minúsculas.
### Declaración XML
**Declaración XML
- <?xml version="1.0" encoding="UTF-8"?>
- **No es obligatoria, pero si aparece tiene que aparecer justo al principio del documento**

**Version**
- Indica la versión de la especificación XML a la que es conforme el documento.

**Encoding**
- Indica la codificación de los caracteres del documento.
- Cuando se utiliza XML para intercambio de información entre un cliente y un servidor, lo más normal es usar codificación UTF-8

### Elementos y atributos
Todo documento debe tener un elemento raíz
- `movies` en el ejemplo

Entre el tag de inicio `(<tag>)` de un elemento y el de fin `(</tag>)` puede haber otros elementos o texto.
