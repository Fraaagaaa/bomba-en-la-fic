## Retrospectiva de la semana 1

**a) ¿Qué ha sido lo mejor de la práctica?**
Lograr una interfaz limpia y funcional sin depender de librerías externas. Ver cómo el diseño *mobile-first* simplifica el CSS final.

**b) ¿Qué fue lo peor?**
El tiempo invertido en ajustar detalles visuales pequeños (centrados, márgenes) que con un framework se hacen automáticos, y asegurar que todos los colores cumplían con el contraste de accesibilidad.

**c) ¿Cuál fue el mejor momento de cada semana durante el trabajo del equipo?**
Cuando conseguimos que la lista de gastos pasara de una columna (móvil) a una tabla/grid (escritorio) correctamente usando solo CSS.

**d) ¿Cuál ha sido el peor?**
Al principio hubo dudas sobre qué etiquetas semánticas eran las más adecuadas para cada parte del informe de gastos 

**e) ¿Qué has aprendido?**
La importancia del HTML semántico no solo para la estructura, sino para la accesibilidad, y cómo las *Media Queries* son la base del diseño responsivo moderno.

**f) ¿Qué necesitáis conservar -como equipo- para las próximas semanas?**
La buena comunicación y la división clara de tareas (Diseño vs. Implementación) que hemos mantenido en esta tarea.

**g) ¿Qué tenéis que mejorar -como equipo- para las próximas semanas?**
Planificar mejor las pruebas de accesibilidad desde el principio, en lugar de dejarlas para el final del desarrollo.

**h) ¿Cómo se relaciona ESTE contenido con otros del curso y con tu titulación?**
Esta práctica sienta las bases del desarrollo Frontend profesional. Se relaciona directamente con la usabilidad (HCI) y con los estándares de ingeniería web que garantizan que el software sea mantenible y accesible universalmente.
