# Informe del Curador-Traductor - Pedro Gómez

## Objetivos de aprendizaje de la semana 1

- **Concepto: HTML Semántico y Estándares Web**
  - **Recursos identificados:** Documentación oficial de MDN Web Docs sobre elementos semánticos (`<section>`, `<article>`, `<nav>`).
  - **Aplicación:** Esencial para cumplir el requisito de no usar `divs` genéricos innecesarios y mejorar la accesibilidad base de la práctica.

- **Concepto: Diseño Responsive y Mobile-First con CSS puro**
  - **Recursos identificados:** Guías de CSS Grid y Flexbox en MDN y CSS-Tricks.
  - **Aplicación:** Sustitución de sistemas de rejilla de frameworks (como Bootstrap) por CSS nativo para lograr que la interfaz de gastos se adapte de móvil a escritorio.

- **Concepto: Accesibilidad Web (WCAG 2)**
  - **Recursos identificados:** W3C Web Accessibility Initiative (WAI) y herramientas de contraste.
  - **Aplicación:** Validación de que la interfaz es perceptible y legible para todos los usuarios.

## Recursos empleados en la semana 1

- **MDN Web Docs (Mozilla Developer Network)**
  - **Descripción:** Referencia principal para HTML5 y CSS3.
  - **Utilidad:** Consultar la compatibilidad de propiedades CSS en el *Baseline* y uso correcto de etiquetas ARIA básicas.

- **W3C Markup Validation Service**
  - **Descripción:** Validador online de código HTML.
  - **Utilidad:** Se utilizó para asegurar que el documento no contenía errores de sintaxis antes de la entrega.

- **WebAIM Contrast Checker**
  - **Descripción:** Herramienta para verificar el contraste de color entre texto y fondo.
  - **Utilidad:** Permitió ajustar los colores de los gastos (rojo/verde) para asegurar que fueran legibles según las normas WCAG AA.
